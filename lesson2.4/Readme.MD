Визначення ієрархії: Створено базовий клас Entity з віртуальним деструктором. Це критично: якщо деструктор не буде віртуальним, видалення new MFP() через вказівник Entity* призведе до витоку пам'яті (деструктори нащадків не викличуться).

Реалізація перевизначення: Класи Scanner та Printer заміщають метод action().

Множинне спадкування: Клас MFP (багатофункціональний пристрій) об'єднує можливості обох. Використано virtual public Entity, щоб у MFP була лише одна копія базового класу.

Демонстрація: У main показано, як вказівник на "щось абстрактне" (Entity*) точно знає, що перед ним саме MFP, і викликає правильну функцію.
